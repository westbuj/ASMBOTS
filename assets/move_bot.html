Moving your asmBot
Moving your bot involves manipulating registers. Your bot is always moving and the direction and speed is determined by the values in the DIR and SPD registers. The FCP accesses these registers and controls the bot's sub-systems.

The FCP recognizes two modes of operation. You can control the mode by setting the NMOD register.

NMODread/write
Mode 0 - The FCP will automatically calculate and set the DIR register causing movement towards the point contained in TRGX, TRGY.

Mode 1 - Your program must set the DIR register. As always, the robot will move in the direction specified in the DIR register.
TRGXread only
Contains the x coordinate of the robot's destination on the soccer field.
If NMOD is set to 1, this registers are ignored. If NMOD is 0, the FCP will set the DIR register for you.
TRGYread only
Contains the y coordinate of the robot's destination on the soccer field.
If NMOD is set to 1, this registers are ignored. If NMOD is 0, the FCP will set the DIR register for you.
BALXread only
Contains the x coordinate of the ball on the soccer field.
BALYread only
Contains the y coordinate of the ball on the soccer field.
BALDread only
Contains the distance to the ball from the robot.
BALRread only
Contains the angle to the ball from the robot.
